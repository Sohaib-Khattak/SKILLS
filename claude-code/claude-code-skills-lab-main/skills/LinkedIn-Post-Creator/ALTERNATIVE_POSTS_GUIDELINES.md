If a user requests a post with a title or specification that indicates a **non-LinkedIn platform or purpose**, you **MUST**:

### Step 1: Identify the Request
Recognize when the requested post is for:
- A different social media platform (Twitter, Instagram, Facebook, TikTok, etc.)
- A different publishing format (Blog post, Email newsletter, Medium article, etc.)
- A different audience context (Internal company memo, press release, etc.)
- Any title or description that deviates from "LinkedIn Post"

### Step 2: Ask for Explicit Permission
Before proceeding, respond with:

```
I notice you're requesting a [PLATFORM/FORMAT] post instead of a LinkedIn post.
This skill is specifically designed for LinkedIn content creation.

Do you want me to:
A) Create a LinkedIn version of this post instead, OR
B) Create the [PLATFORM/FORMAT] post anyway (I will adapt the format and tone accordingly)?

Please confirm your preference before I proceed.
```

### Step 3: Validate User Consent
Only proceed if the user:
- Explicitly confirms they want the alternative format
- Acknowledges this is outside the skill's primary scope
- Provides any additional platform-specific guidelines if needed

### Step 4: Adapt and Create
Once permission is granted:
- Maintain professionalism and quality standards
- Adapt the tone and structure to suit the target platform
- Note the platform/format at the beginning of the output
- Include a disclaimer: "‚ö†Ô∏è This post was created for [PLATFORM] and deviates from the LinkedIn post standard format."
